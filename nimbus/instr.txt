  # Project: Nimbus - Cloud-Based Web IDE

  ## Overview
  Nimbus is a cloud-based web IDE in early development,
  built with **Next.js 16** (App Router) and **Monaco
  Editor**. The current focus is building core editor
  functionality before cloud deployment.

  **Repository**: https://github.com/R7nX/Nimbus.git
  **Version**: 0.1.0

  ---

  ## Tech Stack
  - **Framework**: Next.js 16.1.6 with App Router
  - **UI**: React 19.1.1, Tailwind CSS 3.4.17, Framer
  Motion 12.23
  - **Editor**: Monaco Editor 0.52.2
  (@monaco-editor/react 4.7.0)
  - **Language**: TypeScript 5

  ## Current Features Implemented

  ### 1. Monaco Editor Integration
  - Full syntax highlighting (Python, JS, TS, JSON, HTML,
   CSS)
  - Dark theme (`vs-dark`), 14px font, minimap disabled
  - Real-time content editing with onChange handler

  ### 2. File Management (`useFileManager` hook)
  - **Open**: File System Access API (Chromium) with
  FileReader fallback
  - **Save**: Writes to original location using stored
  file handle
  - **Save As**: Save dialog with new filename
  - **Dirty tracking**: Tracks unsaved changes with
  visual indicator (•)
  - **Language detection**: Auto-infers from file
  extension
  - **Unsaved warning**: beforeunload listener prevents
  accidental navigation

  ### 3. Authentication (UI only)
  - Login/Signup forms with validation
  - Expects external backend at `http://localhost:3000`
  - Endpoints: `POST /login`, `POST /signup`
  - Session persistence via localStorage/sessionStorage

  ### 4. UI Components
  - FileToolbar: New, Open, Save, Remove buttons
  - Header with filename display and dirty indicator
  - Full-screen responsive editor layout

  ---

  ## Architecture Patterns

  - **State Management**: React hooks only (useState,
  useRef) - no Redux/Zustand
  - **File I/O**: Browser File System Access API with
  graceful fallback
  - **Styling**: Tailwind CSS utility classes
  - **Components**: Functional components with TypeScript
  - **Routing**: Next.js App Router (file-based)

  ---

  ## What's NOT Implemented Yet

  - Backend/API routes (auth expects external server)
  - Cloud file storage/sync
  - Project/folder management (multi-file support)
  - Integrated terminal
  - Real-time collaboration
  - Database integration
  - File tree/explorer sidebar

  ---

  ## Development Guidelines

  1. **Import alias**: Use `@/*` for imports (maps to
  root)
  2. **Styling**: Use Tailwind classes, dark theme
  preferred (neutral-900/800)
  3. **File operations**: Extend useFileManager hook for
  new functionality
  4. **New pages**: Add to `app/` directory following App
   Router conventions
  5. **Components**: Place reusable components in
  `components/`
  6. **Hooks**: Place custom hooks in `components/hooks/`

  ---

  ## Browser Support
  - **Full**: Chrome, Edge, Opera (File System Access
  API)
  - **Limited**: Safari, Firefox (no native save, uses
  download fallback)
